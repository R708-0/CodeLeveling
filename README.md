# CODE LEVELING
#### Video Demo:  https://youtu.be/CNj7FyWwOxg
#### Description:

Este proyecto es una mezcla de una aplicación de tareas, un portfolio y un sistema de juego para aprender programacion diseñado para ayudar a personas que están aprendiendo programación a mantener la motivación, la disciplina y el enfoque a largo plazo. A diferencia de un portfolio tradicional, que solo muestra proyectos finalizados, esta plataforma convierte el proceso de aprendizaje en un sistema de progreso continuo, donde cada acción cuenta y cada pequeño avance se traduce en crecimiento visible.

La idea central es que el usuario no solo tenga un espacio donde mostrar sus proyectos y habilidades, sino que también pueda vivir un proceso de progresión, similar al de un videojuego. Cada tarea completada, cada habilidad practicada y cada proyecto agregado se convierte en una oportunidad para ganar puntos de experiencia (XP) y subir de nivel. De esta manera, el aprendizaje deja de ser una actividad lineal y se convierte en una aventura interactiva.

Basandome en método pomodoro la aplicación tambien esta pensada para organizar el estudio de manera cíclica con un tiempo de atención minimo en cada ciclo que sirve para prevenir el agotamiento mental, sin embargo cada ciclo esta pensado como una progresión lineal de un metodo de estudio general.  

## Motivación para crear el proyecto

El aprendizaje se ha visto profundamente afectado por un problema cada vez más común: el déficit de atención provocados por la sobreexposición a redes sociales, el consumo excesivo de contenido rápido y la falta de estructuras claras.

Muchas personas interesadas en aprender programación abandonan sus estudios porque sienten que el proceso es demasiado largo, poco estimulante o carente de recompensas inmediatas. 
esto causa que:

Se distraigan fácilmente

Abandonen proyectos a mitad de camino

No logren consolidar hábitos de estudio

Sientan frustración al no ver resultados inmediatos

Este proyecto ofrece una solución práctica: introducir elementos de juego que generen satisfacción, refuercen la constancia y conviertan cada acción tiene una recompensa, cada meta es alcanzable y el progreso es medible, visible y estimulante.

## Objetivos del proyecto

- Facilitar el aprendizaje de programación mediante un entorno visual y dinámico.
- Motivar la constancia a través de recompensas inmediatas como XP, niveles y logros.
- Ofrecer un portfolio interactivo que muestre habilidades y proyectos de manera atractiva.
- Convertir el estudio en una experiencia lúdica, reduciendo la sensación de monotonía.
- Atender a las necesidades de quienes tienen déficit de atención, ofreciendo estímulos claros y metas alcanzables.
- Ayudar a la organización metódica para el entendimiento de un tema en especifico. 
- Incremeentar la capacidad de aprendizaje autónomo.
- Incentivar la aplicación teórica, práctica y experimental del conocimiento.

Es un portfolio que cuenta una historia, mostrando no solo el resultado final, sino el camino recorrido para llegar hasta ahí.

## Caracteristicas principales del proyecto

El sistema se estructura en tres pilares fundamentales. 

- Perfil de usuario: Cada persona cuenta con un perfil donde se muestran sus datos.
    - Foto de perfil (si no subió ningun archivo se utiliza una fotografía estandar)
    - Nombre completo. 
    - Nivel.
    - Titulo y rango.
    - Progreso de la experiencia obtenida.
    - Descripción del usuario (Descripción estandar si usted no ingreso una)

- Sistema de habilidades: Las habilidades se representan como barras de progreso. Al completar tareas relacionadas, el usuario gana XP y aumenta su nivel en esa habilidad específica. Por ahora existe un catalogo de habilidades relacionados al curso cs50 que son:
    - Python.
    - JavaScript.
    - Sql.
    - Html.
    - Css.

- Gestión de tareas: Las tareas se presentan como objetivos que, al completarlas otorgan puntos de experiencia a la habilidad vinculada o al usuario directamente, como es el caso de las tareas de proyectos.

Las tareas de habilidad se dividen en 3 sub tareas cada una.
    - 20 minutos de teoría 
    - 20 minutos de practica
    - 20 minutos de experimentación 
Las tareas pueden reiniciarse para cumpir otro ciclo de tareas y seguir subiendo de nivel.

Basado en el metodo Pomodoro, se espera espera que el usuario siga una líne logica para el aprendizaje, donde los primeros 20 minutos se dedicara a reunir informacion teórica, los siguientes 20 minutos se dedicará a ponerla en práctica y los ulitmos 20 minutos buscará alternativas y experimentará de forma creativa sobre lo aprendido.

Este metodo de estudio busca reducir en cansancio mental, asi como también redirigir el deficit de atención a un solo objetivo dividido en 3 fases distintas que harán del estudio algo mas dinámico y menos tedioso.

## Conclusión

Este proyecto no es solo una herramienta técnica, sino también una propuesta pedagógica y motivacional quwebusca transformar la manera en que los estudiantes de programación enfrentan sus retos, ofreciendo un entorno que combina aprendizaje, juego y progreso personal. En tiempos donde la atención es un recurso escaso, este proyecto se convierte en un aliado para mantener la constancia y alcanzar metas de manera divertida y significativa.
